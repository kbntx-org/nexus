# Onboarding overview

This section is the fastest path for a new engineer to become productive in Nexus.

## Goals

By the end of onboarding, you should be able to:

1. Understand where code lives and what deploys where.
2. Run checks locally before opening a PR.
3. Know how platform components are reconciled with GitOps.

## Prerequisites

- Node.js + pnpm
- Docker (for local docs and image workflows)
- kubectl + helm (for chart troubleshooting)

## Suggested path

1. Read [Repository map](../reference/repository-map.md).
2. Follow [Local development](local-development.md).
3. Complete [First week checklist](first-week-checklist.md).
